## 🧰 Мемоизация и производительность

### Микро-синтаксис: Мемоизация компонента (React.memo)
Предотвращает лишние перерендеры дочернего компонента, если его входные данные (`props`) не изменились.

```tsx
import React from 'react';

type Props = { items: string[] };

// 1. Обычный чистый компонент отображения
const ListComponent = ({ items }: Props) => {
  return <ul>{items.map(item => <li key={item}>{item}</li>)}</ul>;
};

// 2. Экспорт компонента, обернутого в React.memo
export const MemoizedList = React.memo(ListComponent);
```
### Микро-синтаксис: Кэширование вычислений (useMemo)
Используется для тяжелых расчетов (например, фильтрации больших массивов). 
Код внутри выполнится повторно только при изменении зависимостей в массиве.

```tsx
// Фильтрация сработает только при изменении items или query
const filteredItems = useMemo(() => {
  return items.filter(item => item.includes(query));
}, [items, query]);
```

### Микро-синтаксис: Стабилизация функций (useCallback)
Сохраняет постоянную ссылку на функцию между перерендерами компонента.
Помогает избежать перерисовок дочерних компонентов, принимающих эту функцию через пропсы.

```tsx
// Ссылка на функцию создается один раз при монтировании
const handleDelete = useCallback((id: number) => {
  setItems((prev) => prev.filter(item => item.id !== id));
}, []); // Массив зависимостей пуст
```
