# 🧩 Моя база паттернов проектирования (React + TypeScript)

Добро пожаловать в структурированную базу знаний готовых кодовых решений. Все паттерны распределены по изолированным файлам в зависимости от их архитектурной роли, оптимизированы под строгие стандарты TypeScript и снабжены наглядными ментальными моделями на русском языке.

---

## 🔍 Инструкция по быстрому и удобному поиску

Для максимального удобства **описание проблемы и ментальные модели написаны на русском языке**, а **все заголовки, ключевые слова и теги для поиска — на английском**.

### Сценарий 1: Поиск в среде разработки (Внутри VS Code)
Это самый быстрый способ найти решение, не отвлекаясь от написания кода:
1. Нажми комбинацию клавиш **`Ctrl + Shift + F`** (Глобальный поиск по всему проекту).
2. Чтобы искать только внутри базы паттернов, в поле **files to include** впиши `./patterns`.
3. Вводи нужный английский тег (например, `xor`, `state-lifting`, `reset`, `select`).
4. В левой панели VS Code мгновенно отобразятся русские заголовки и куски кода. Кликни, скопируй и вставь в рабочий файл.

### Сценарий 2: Поиск на сайте GitHub
Если базы нет под рукой на рабочем компьютере:
1. Зайди в этот репозиторий и нажми клавишу **`/`** (слэш) или комбинацию **`Ctrl + K`**.
2. В строке умного поиска GitHub Blackbird введи ключевое слово (например, `data-enrichment` или `onblur`).
3. Движок мгновенно подсветит нужную строчку в конкретном файле.

## 📂 Карта репозитория и навигация

Вся кодовая база вынесена в папку `patterns/`. Каждый паттерн внутри файлов визуально отделен линиями и обёрнут в стандартные блоки сворачивания VS Code (`//#region` и `//#endregion`), что позволяет сворачивать их в аккуратное интерактивное оглавление с помощью горячих клавиш `Ctrl + K, Ctrl + 0`.

### 💻 [Интерфейс и Формы (patterns/react-ui.tsx)](./patterns/react-ui.tsx)
*Управление UI-компонентами, инпутами, выпадающими списками, событиями фокуса и валидацией.*
* **`Controlled Select Component`** — Выпадающие списки, обработка `onChange` и безопасный рендеринг через `.map()`.
* **`onBlur + setTimeout (Focus Management)`** — Задержка закрытия модальных окон/дропдаунов для корректного срабатывания кликов.
* **`Reset Fields and Errors`** — Синхронный сброс стейтов формы и скрытие предупреждений об ошибках при сабмите.

### 🧠 [Архитектура и Стейт (patterns/react-state.tsx)](./patterns/react-state.tsx)
*Логика управления состоянием React, иммутабельные мутации массивов и синхронизация данных.*
* **`Data Enrichment (Merging Arrays)`** — Безопасное склеивание плоских массивов с сервера через `.map()` и `.find()` без использования опасного оператора `!`.
* **`Lifting State Up & Safe ID Generation`** — Поднятие состояния от ребенка к родителю и безопасный расчет `maxId + 1` через `.reduce()` без риска переполнения стека.

### ⚡ [Чистый TypeScript / Алгоритмы (patterns/ts-utils.ts)](./patterns/ts-utils.ts)
*Изолированные функции-хелперы, чистые алгоритмы и оптимизация вычислений, не привязанные к JSX.*
* **`Find Single Unique Number (XOR)`** — Побитовый ультимативный поиск одиночного числа в массиве дубликатов со скоростью $O(n)$ и памятью $O(1)$.
* **`Find Single Unique Number (Indexes)`** — Декларативный способ поиска через `indexOf === lastIndexOf` (с анализом производительности).
