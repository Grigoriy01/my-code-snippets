## 🧩 Паттерн: Объединение двух массивов (Обогащение данных)

### ❓ Проблема
С сервера прилетает плоский массив (например, задачи `todos`), где есть только `userId`. А интерфейсу или типам TypeScript нужен глубокий объект, где внутри каждой задачи лежит полноценный объект автора `user`. TypeScript ругается на несовпадение типов.

### 🧠 Как это понять (Ментальная модель)
У нас есть коробка с письмами (задачи) и список жителей дома (пользователи). На письме написан только номер квартиры (`userId`). Мы берем каждое письмо, смотрим на номер квартиры, ищем этого человека в списке жителей и "вкладываем" его визитку прямо внутрь конверта с письмом. Теперь у нас коробка писем, в каждом из которых уже лежит инфо об авторе.

### 💻 Код-рецепт (React + TypeScript)

```typescript
// 1. Импортируем типы и сырые данные
import { Todo, User } from './types';
import { todosFromServer, usersFromServer } from './api';

// 2. Склеиваем массивы через: 
 .map() и .find() перед созданием стейта
const preparedTodos: Todo[] = todosFromServer.map(todo => {
  // Ищем юзера, у которого id совпадает с userId в задаче
  const foundUser = usersFromServer.find(user => user.id === todo.userId);

  return {
    ...todo,           // Копируем все старые поля задачи (id, title, completed)
    user: foundUser!   // Добавляем объект юзера ("!" говорит TS, что юзер точно есть)
  };
});

// 3. Передаем готовую базу в useState
const [todos, setTodos] = useState<Todo[]>(preparedTodos);
```

## 🧩 Паттерн: Поиск уникального (одиночного) элемента в массиве дубликатов

### ❓ Проблема
Дан массив чисел, где каждый элемент повторяется дважды, и только одно число встречается один раз. Нужно найти это уникальное число. Обычный перебор через вложенные циклы работает медленно, а использование промежуточных объектов-хранилищ требует выделения дополнительной памяти. Хочется решить задачу либо максимально эффективно по памяти и скорости, либо простым встроенным методом.

### 🧠 Как это понять (Ментальная модель)
<details>
<summary>**Вариант 1 (Магия XOR / Исключающее ИЛИ/ Оператор `^`):**</summary>
 
 Представь, что числа — это люди, и у каждого есть свой точный близнец. На дискотеке они танцуют парами. Оператор `^` (XOR) работает как зеркало-аннигилятор: когда два одинаковых человека сталкиваются, они исчезают (`5 ^ 5 = 0`). А если число сталкивается с нулем, оно остается неизменным (`5 ^ 0 = 5`). Если мы запустим всех в одну комнату (пропустим через `reduce`), все пары аннигилируют друг друга в ноль, и в комнате останется стоять только один одинокий человек без пары.
</details>
<details>
 <summary>**Вариант 2 (Поиск по индексам):**</summary>
 
 Представь шеренгу людей. Мы ищем человека, который пришел один. Мы спрашиваем у шеренги: «С какого края стоит первый человек в такой куртке?» (это `indexOf`), а затем: «А с какого края стоит последний в такой куртке?» (`lastIndexOf`). Если номера позиций совпали, значит, этот человек в шеренге всего один.
</details>


### 💻 Код-рецепт (React + TypeScript)

```typescript
// Способ 1: Оптимальный (Линейное время O(n), память O(1)) через побитовый оператор XOR
export const findSingleNumXOR = (nums: number[]): number => {
  // acc аккумулирует результат, n — текущее число. 
  // Одинаковые числа при XOR дают 0, в остатке останется уникальное число.
  return nums.reduce((acc, n) => acc ^ n, 0);
};

// Способ 2: Декларативный (Через встроенные методы массивов)
export const findSingleNumIndexes = (nums: number[]): number | undefined => {
  // Ищем элемент, у которого первый индекс вхождения равен последнему индексу вхождения
  return nums.find(n => nums.indexOf(n) === nums.lastIndexOf(n));
};

// Пример использования в React-компоненте
import React, { useState } from 'react';

export const UniqueNumberChecker: React.FC = () => {
  const [numbers] = useState<number[]>([4, 1, 2, 1, 2]);
  
  // Применив первый способ, получим 4 (так как 1 и 2 уничтожатся)
  const uniqueNum = findSingleNumXOR(numbers);

  return (
    <div>
      <p>Исходный массив: {JSON.stringify(numbers)}</p>
      <p>Уникальное число: <strong>{uniqueNum}</strong></p>
    </div>
  );
};
```
## 🔼 Паттерн: Поднятие состояния (Lifting State Up) + Деструктуризация аргументов + найти максимальный ID в массиве объектов + new ID +1

### ❓ Проблема
Дочерний компонент (например, форма `NewTodoForm`) собирает «сырые» данные от пользователя (текст инпута, выбор в селекте). Но у формы нет доступа к общему стейту со списком задач и пользователей, она не может сама добавить элемент в базу. 

### 🧠 Как это понять (Ментальная модель)
Форма — это курьер. Её задача — просто взять у клиента данные (имя и заказ), запаковать в один пакет (объект) и принести в главный офис (родительский компонент `App`). Главный офис принимает этот пакет, распаковывает (деструктуризация), ставит на документ печать, дату и ID (модернизация) и заносит в общую амбарную книгу (обновляет стейт).

### 💻 Код-рецепт (React + TypeScript)

**1. В родительском компоненте (`App.tsx`):**
```typescript
const addTodo = ({ title, userId }: { title: string; userId: number }) => {
  // 1. Находим максимальный ID среди текущих задач
  // Если массив вдруг пустой, подставляем 0, чтобы 0 + 1 дал первый ID
  const maxId = todos.length > 0 
    ? Math.max(...todos.map(todo => todo.id)) 
    : 0;

  const nextId = maxId + 1; // Твой инкремент (ID + 1)

  // 2. Находим пользователя для обогащения данных
  const currentUser = users.find(user => user.id === userId);

  // 3. Формируем полноценный объект новой задачи
  const newTodo: Todo = {
    id: nextId, // Передаем вычисленный ID
    title,
    userId,
    completed: false,
    user: currentUser!
  };

  // 4. Обновляем стейт, добавляя новую задачу в конец (или в начало) списка
  setTodos(prevTodos => [...prevTodos, newTodo]);
};
```

## 🧩 Паттерн: Задержка закрытия окна при потере фокуса (onBlur + setTimeout)

### ❓ Проблема
<details>
 
<summary>**При создании кастомных выпадающих списков (Dropdown)</summary>

 окон автокомплита или подсказок мы используем событие `onBlur` (потеря фокуса), чтобы автоматически закрывать окно, когда пользователь кликает вне его области. Но возникает баг: если пользователь кликает по элементу *внутри* самого выпадающего списка (например, хочет выбрать пункт меню), событие `onBlur` срабатывает **быстрее**, чем событие `onClick` на этом пункте. В итоге окно исчезает из DOM, а клик по пункту так и не регистрируется.
</details>

### 🧠 Как это понять (Ментальная модель)
<details>
<summary>Представь охранника (событие `onBlur`)</summary>
 
 который стоит у дверей клуба (нашего выпадающего списка). Как только последний посетитель отворачивается от двери, охранник моментально выключает свет и запирает клуб. Посетитель хотел нажать на кнопку внутри, но не успел — стало темно. `setTimeout` на 200 миллисекунд — это как если бы мы попросили охранника: «Когда увидишь, что все уходят, не закрывай дверь мгновенно. Сосчитай про себя до трех». За эти «три секунды» (200 мс) посетитель как раз успевает нажать на нужную кнопку, клик срабатывает, а уже потом закрывается сама дверь.
</details>

### 💻 Код-рецепт (React + TypeScript)

**В дочернем компоненте (например, `SearchAutocomplete.tsx`):**
```typescript
import React, { useState } from 'react';

export const SearchAutocomplete: React.FC = () => {
  const [isActive, setIsActive] = useState<boolean>(false);
  const [selectedItem, setSelectedItem] = useState<string>('');
  const items = ['React', 'TypeScript', 'JavaScript'];

  return (
    <div style={{ position: 'relative' }}>
      {/* Инпут, который открывает список при фокусе и закрывает при потере фокуса */}
      <input
        type="text"
        placeholder="Поиск технологии..."
        value={selectedItem}
        onChange={(e) => setSelectedItem(e.target.value)}
        onFocus={() => setIsActive(true)}
        
        // Паттерн: откладываем закрытие, чтобы onClick внутри списка успел отработать
        onBlur={() => {
          setTimeout(() => {
            setIsActive(false);
          }, 200); // 200мс достаточно для регистрации клика на элементах списка
        }}
      />

      {/* Выпадающий список, который рендерится в зависимости от стейта */}
      {isActive && (
        <ul style={{ position: 'absolute', top: '100%', left: 0, background: '#fff', border: '1px solid #ccc' }}>
          {items.map((item) => (
            <li 
              key={item} 
              onClick={() => {
                // Если убрать setTimeout в onBlur, этот код никогда не выполнится
                setSelectedItem(item);
                console.log(`Выбрано: ${item}`);
              }}
              style={{ cursor: 'pointer', padding: '8px' }}
            >
              {item}
            </li>
          ))}
        </ul>
      )}
    </div>
  );
};
```
