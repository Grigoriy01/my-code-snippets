# TS-React
## 🧩 Паттерн: Объединение двух массивов (Обогащение данных)

### ❓ Проблема
С сервера прилетает плоский массив (например, задачи `todos`), где есть только `userId`. А интерфейсу или типам TypeScript нужен глубокий объект, где внутри каждой задачи лежит полноценный объект автора `user`. TypeScript ругается на несовпадение типов.

### 🧠 Как это понять (Ментальная модель)
У нас есть коробка с письмами (задачи) и список жителей дома (пользователи). На письме написан только номер квартиры (`userId`). Мы берем каждое письмо, смотрим на номер квартиры, ищем этого человека в списке жителей и "вкладываем" его визитку прямо внутрь конверта с письмом. Теперь у нас коробка писем, в каждом из которых уже лежит инфо об авторе.

### 💻 Код-рецепт (React + TypeScript)

```typescript
// 1. Импортируем типы и сырые данные
import { Todo, User } from './types';
import { todosFromServer, usersFromServer } from './api';

// 2. Склеиваем массивы через: 
 .map() и .find() перед созданием стейта
const preparedTodos: Todo[] = todosFromServer.map(todo => {
  // Ищем юзера, у которого id совпадает с userId в задаче
  const foundUser = usersFromServer.find(user => user.id === todo.userId);

  return {
    ...todo,           // Копируем все старые поля задачи (id, title, completed)
    user: foundUser!   // Добавляем объект юзера ("!" говорит TS, что юзер точно есть)
  };
});

// 3. Передаем готовую базу в useState
const [todos, setTodos] = useState<Todo[]>(preparedTodos);
------------------------------
## 🧩 Паттерн: Поиск уникального (одиночного) элемента в массиве дубликатов

### ❓ Проблема
Дан массив чисел, где каждый элемент повторяется дважды, и только одно число встречается один раз. Нужно найти это уникальное число. Обычный перебор через вложенные циклы работает медленно, а использование промежуточных объектов-хранилищ требует выделения дополнительной памяти. Хочется решить задачу либо максимально эффективно по памяти и скорости, либо простым встроенным методом.

### 🧠 Как это понять (Ментальная модель)
**Вариант 1 (Магия XOR / Исключающее ИЛИ):** Представь, что числа — это люди, и у каждого есть свой точный близнец. На дискотеке они танцуют парами. Оператор `^` (XOR) работает как зеркало-аннигилятор: когда два одинаковых человека сталкиваются, они исчезают (`5 ^ 5 = 0`). А если число сталкивается с нулем, оно остается неизменным (`5 ^ 0 = 5`). Если мы запустим всех в одну комнату (пропустим через `reduce`), все пары аннигилируют друг друга в ноль, и в комнате останется стоять только один одинокий человек без пары.

**Вариант 2 (Поиск по индексам):** Представь шеренгу людей. Мы ищем человека, который пришел один. Мы спрашиваем у шеренги: «С какого края стоит первый человек в такой куртке?» (это `indexOf`), а затем: «А с какого края стоит последний в такой куртке?» (`lastIndexOf`). Если номера позиций совпали, значит, этот человек в шеренге всего один.

### 💻 Код-рецепт (React + TypeScript)

```typescript
// Способ 1: Оптимальный (Линейное время O(n), память O(1)) через побитовый оператор XOR
export const findSingleNumXOR = (nums: number[]): number => {
  // acc аккумулирует результат, n — текущее число. 
  // Одинаковые числа при XOR дают 0, в остатке останется уникальное число.
  return nums.reduce((acc, n) => acc ^ n, 0);
};

// Способ 2: Декларативный (Через встроенные методы массивов)
export const findSingleNumIndexes = (nums: number[]): number | undefined => {
  // Ищем элемент, у которого первый индекс вхождения равен последнему индексу вхождения
  return nums.find(n => nums.indexOf(n) === nums.lastIndexOf(n));
};

// Пример использования в React-компоненте
import React, { useState } from 'react';

export const UniqueNumberChecker: React.FC = () => {
  const [numbers] = useState<number[]>([4, 1, 2, 1, 2]);
  
  // Применив первый способ, получим 4 (так как 1 и 2 уничтожатся)
  const uniqueNum = findSingleNumXOR(numbers);

  return (
    <div>
      <p>Исходный массив: {JSON.stringify(numbers)}</p>
      <p>Уникальное число: <strong>{uniqueNum}</strong></p>
    </div>
  );
};
