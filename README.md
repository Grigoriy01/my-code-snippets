# 🧰 Моя база знаний: Архитектурное LEGO (React & TypeScript)

Добро пожаловать в репозиторий атомарных паттернов и микро-синтаксиса. Здесь нет громоздкого UI-мусора, стилей и верстки. Только чистая механика кода, разбитая на изолированные инструменты, которые можно скопировать и внедрить в проект за 3 секунды.

---

## 🗺️ Карта репозитория

### 1. 🌐 [Асинхронные эффекты и API](./patterns/async-effects.md)
*Все, что связано с жизненным циклом хуков, сетевыми запросами и асинхронным поведением.*
* **Паттерн: Безопасный useEffect** (`#use-effect` `#race-condition` `#loading`)
  * Готовый скелет загрузки данных с индикатором (`isLoading`), обработкой ошибок и встроенным флагом отмены (`isCurrentRequest`) для защиты от Race Condition.
* **Микро-синтаксис: Сброс стейта при смене пропса** (`#state-reset`)
  * Правильная превентивная очистка старого состояния до начала нового триггера.
* **Микро-синтаксис: Безопасный Fetch-сервис** (`#fetch` `#typescript`)
  * Обертка сетевого слоя с ручной генерацией исключений для HTTP-ошибок (404/500).
* **Микро-синтаксис: Перехват ошибок `unknown`** (`#error-handling`)
  * Безопасное TypeScript-приведение типов ошибок через проверку `instanceof Error`.

### 2. ⚡ [Мемоизация и производительность](./optimization.md)
*Инструменты для кэширования и контроля лишних перерендеров приложения.*
* **Микро-синтаксис: React.memo** (`#memo` `#components`)
  * Поверхностное сравнение пропсов для блокировки холостых перерисовок дочерних компонентов.
* **Микро-синтаксис: useMemo** (`#use-memo` `#cache`)
  * Кэширование тяжелых вычислений (фильтрации, сортировки) до изменения делендеров.
* **Микро-синтаксис: useCallback** (`#use-callback` `#functions`)
  * Стабилизация ссылок на функции-колбэки для сохранения иммунитета дочерних элементов.

### 3. 👥 [Управление состоянием](./state-management.md)
*Архитектурные подходы распределения бизнес-логики между компонентами.*
* **Паттерн: Поднятие состояния (Lifting State Up)** (`#architecture` `#state-flow`)
  * Разделение на "глупую" форму ввода (сбор сырого payload) и "умного" родителя-контроллера (валидация, генерация ID, иммутабельное обновление стейта).

### 4. 🎨 [Интерактивные UI-компоненты](./ui-patterns.md)
*Решение классических интерфейсных багов и стыковки браузерных событий.*
* **Паттерн: onBlur + setTimeout** (`#event-loop` `#dropdown` `#autocomplete`)
  * Синхронизация Race Condition событий потери фокуса и клика в кастомных выпадающих списках. Искусственный сдвиг закрытия окна в очередь макрозадач.

### 5. 🧮 [Трансформация массивов и данных](./array-helpers.md)
*Чистые хелперы и алгоритмические трюки для обработки JS-структур.*
* **Паттерн: Обогащение данных (Data Enrichment)** (`#arrays` `#data-join`)
  * Склейка плоских массивов по внешнему ID с использованием безопасного дефолтного фолбека (защита рантайма от `undefined`).
* **Паттерн: Вычисление инкрементного ID** (`#reduce` `#algorithms`)
  * Нахождение максимального ID через линейный `reduce` $O(n)$ с защитой от переполнения стек-вызовов (Call Stack Length).
* **Алгоритм: Поиск уникального элемента через XOR** (`#bitwise` `#performance`)
  * Поиск единственного числа без пары за линейное время $O(n)$ и с константной памятью $O(1)$ при помощи побитового оператора `^`.

## 🚀 Как пользоваться базой
1. Выбираешь нужную категорию в списке выше.
2. Переходишь в `.md` файл.
3. Читаешь краткую **ментальную модель** (если забыл, как работает концепт).
4. Копируешь атомарный блок кода и адаптируешь имена переменных под свой проект.
*База знаний спроектирована с возможностью бесконечного масштабирования. При добавлении нового паттерна обязательно используй утвержденный системный промт генерации.*
